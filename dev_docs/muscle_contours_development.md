# Muscle Contours 独立轮廓模块

## 目标

`SlicerMuscleContours` 用于在二维医学图像的多个关键层绘制可编辑、平滑的闭合轮廓，在三维视图中检查全部空间轮廓，并通过关键层之间的有符号距离场插值生成分割体积。

该模块与 `SlicerNNInteractive` 的 AI 提示节点隔离。轮廓是持久 MRML 数据，保存场景后可以继续编辑，不会被推理提示清理流程删除。

## 当前 MVP

- 面板顶部“更新模块”调用 `slicer.util.reloadScriptedModule` 热重载 Python
  代码和界面，不关闭 Slicer，也不清空当前 MRML 场景数据。
- 每个关键层使用一个 `vtkMRMLMarkupsClosedCurveNode`。
- 绘制由模块接管二维视图鼠标事件：每次左键单击只生成一个控制点，
  鼠标移动不采样，右键结束闭合。
- “绘制视图”可明确选择 Red、Yellow 或 Green。鼠标仍位于二维视图时优先
  使用鼠标所在视图并同步下拉框；从模块面板启动时使用下拉框选择，三种
  视图共享相同的逐点绘制与平面约束逻辑。
- “绘制视图”和“调整轮廓所在视图”使用相同索引顺序并双向同步；任一
  下拉框改变时阻断目标框信号后设置相同索引，避免循环触发。
- 绘制模式监听所属视图的切片偏移量；真实翻页超过 `1e-4` mm 时自动关闭。
  已有至少 3 点的轮廓保留，不足 3 点的轮廓取消。单键快捷键为 `D`；
  文本框获得焦点时不响应快捷键，避免影响名称输入。
- 控制点使用 Cardinal Spline 形成平滑闭合曲线。
- “插入控制点”模式允许在选中曲线附近单击，将新点插入对应控制点区间，
  插入后可继续拖动微调。
- 控制点拖动后自动投影回创建轮廓时的切片平面。
- 关闭 Markups 整体变换手柄，不显示每层轮廓中心的平移/旋转轮盘；二维
  控制点本身仍可单独拖动。
- 二维视图只显示与当前切片重合的轮廓。
- 三维视图显示全部轮廓线和只读控制点。控制点不能在 3D 中拖动，但可点击：
  模块以屏幕空间 12 像素容差拾取最近点，随后选中所属轮廓并跳到对应二维
  层面。控制点位置编辑仅在二维切片中进行。每个轮廓使用独立的 2D/3D
  Markups 显示节点实现。
- 支持与主模块一致的单序列定位线拖拽旋转：抓取二维视图中的另一切片
  彩色交线，围绕三平面交点旋转目标切片。
- Red、Yellow、Green 切片在三维视图中显示为带彩色边框的半透明定位面，
  实时跟随滚动和旋转；面板可统一控制显隐及填充亮度/透明度。
- 轮廓列表与视图双向联动：在三维中点击只读控制点时，自动选中对应轮廓
  条目并跳到其保存的层面；点击列表中的轮廓条目时，对应来源视图
  通过轮廓保存的 RAS 平面原点跳转到该层。
- 层面联动与绘制交互严格隔离：只响应 3D 自定义点拾取和用户手动点击列表；
  不监听二维 Markups 控制点交互，不设置 `ActivePlaceNode`。绘制、复制或
  添加调整轮廓时仅更新列表选中状态，不触发导航。
- 用户翻动某个二维视图后，如果列表当前选中的轮廓属于该视图且新切片已
  离开轮廓保存平面（距离大于 0.25 mm 或方向不再平行），自动取消列表选中；
  其他视图的 ModifiedEvent 不影响当前选择。
- 二维可见性与节点锁定同步：轮廓只添加到其创建来源视图的 ViewNodeID；
  仅当该视图与保存平面平行且距离小于 0.25 mm 时解锁。翻页后曲线隐藏并
  立即锁定，防止 Slicer 拾取器命中不可见的旧层控制点。
- 删除轮廓时同时清理其辅助 3D Markups 显示节点，并从当前输出分割中删除
  同名肌肉 Segment，避免保留与关键轮廓不一致的旧三维表面；其他肌肉不受
  影响，剩余轮廓需要时可重新生成。
- Segment 颜色按“Segmentation 节点 + 肌肉名称”持久化在节点属性中。同一
  输出分割内同名肌肉重新生成或删除后再生成时复用颜色；若现有 Segment
  被用户手动改色则优先继承。新的 Segmentation 节点使用其 MRML ID 与名称
  生成新的稳定高饱和颜色，同一节点内不同肌肉仍分别着色。
- 每个轮廓通过节点引用 `SlicerMuscleContours.OutputSegmentation` 归属于
  一个输出 Segmentation。列表、3D 控制点拾取、复制、调整轮廓和生成均只
  读取当前输出节点的轮廓；切换或新建 Segmentation 得到独立工作集，不删除
  或改写旧输出。旧版无归属引用的轮廓在模块首次加载时绑定到当时选中的
  输出 Segmentation。
- 输出模型管理支持旧工程节点：可按当前输出 Segmentation 统一显示/隐藏
  三维表示；“删除当前输出的全部 3D 结果”经确认后删除其中所有 Segment，
  但保留输出节点、颜色属性和所属闭合轮廓，允许稍后重新生成。
- 支持把最近/选中的轮廓平移复制到当前切片。
- 同一肌肉至少两个关键层可通过二维 SDF 插值生成 Segment 和闭合表面。
- 生成分割后，可选择 Red/Yellow/Green 视图并在当前中间层添加调整轮廓：
  该功能只用当前 `SliceToRAS` 平面截取 Segment 闭合表面，不修改参考影像、
  切片方向或视图背景。截取的最大闭合边界重采样为 24 个可拖动控制点，
  调整后重新生成即可把该层作为新的关键层约束。
- 模块入口：`slicer_plugin/SlicerMuscleContours/`。

## 数据属性

每个受管轮廓设置以下属性：

- `SlicerMuscleContours.Managed`
- `SlicerMuscleContours.Group`
- `SlicerMuscleContours.SourceView`
- `SlicerMuscleContours.SliceIndex`
- `SlicerMuscleContours.PlaneOriginRAS`
- `SlicerMuscleContours.PlaneNormalRAS`
- `SlicerMuscleContours.IJKAxis`
- 节点引用 `SlicerMuscleContours.ReferenceVolume`

## MVP 限制

- 轴对齐和斜切面都可以绘制、显示、编辑和生成体积。
- 轴对齐轮廓使用逐层二维 SDF 插值；平行斜切轮廓使用闭合曲线统一重采样、
  点序对齐、三角面放样和参考体素网格栅格化。
- 一组斜切关键层必须互相平行（夹角误差不超过 2.5°）；不支持在绘制途中
  改变旋转角度后继续混合插值。
- 同一肌肉可同时使用多个正交视图：按参考影像 IJK 轴分组独立生成候选
  掩膜，再做多数投票融合。两个方向取交集；三个方向保留至少两个方向同意
  的体素。某个轴对齐方向只有一个轮廓时，沿法向扩展为轮廓柱参与空间约束；
  两个以上层面使用正常插值。单个斜切轮廓仍要求补画同方向第二层。若严格
  多数/交集融合为空，则自动回退到并集融合并写入警告日志，不再阻止生成。
- 一次生成只支持一个切片方向，不混合轴向、冠状和矢状轮廓。
- 轮廓在三维视图中可选择和拖动，但始终被约束在原平面；暂不支持整体改换所属层。
- 当前只生成单个肌肉 Segment；多肌肉共享边界和冲突解决尚未实现。
