# SlicerNNInteractive 插件功能说明

> 本文档覆盖上游 nnInteractive 核心功能 + 全部二次开发功能。
> 实现全部在客户端 `slicer_plugin/SlicerNNInteractive/SlicerNNInteractive.py`；
> 服务端 `server/` 与上游保持一致，未添加任何新接口。

---

## 一、基础架构

### 1.1 Client-Server 分离

| 组件 | 职责 |
|------|------|
| **服务端** (`server/`) | FastAPI + nnInteractive 推理会话；需要 NVIDIA GPU（推荐 ≥10 GB VRAM）；单会话有状态，同一时刻只能服务一个客户端 |
| **客户端** (`slicer_plugin/`) | 3D Slicer 扩展；负责所有 UI、交互路由、几何转换、同步、布尔编辑；与服务端通过 HTTP 通信 |

### 1.2 自动同步机制（`@ensure_synched`）

每次发送提示之前，装饰器自动检查：
1. 推理图像是否变化 → 变化则重传图像（`upload_image_to_server`）
2. 当前分割掩膜是否变化 → 变化则重传分割（`upload_segment_to_server`）
3. 服务端报 "No image uploaded" 时，会透明重传后重试

用户在 Slicer 内用任何方式修改了分割，下一次 AI 提示都会自动把最新状态同步到服务端，无需手动操作。

### 1.3 持久化偏好（QSettings）

以下设置跨 Slicer 会话保存：
- 服务端 URL、滚轮吸附、段不透明度默认值
- 高分辨率输出开关 + 间距、平滑插值开关、显示平滑开关 + 强度
- 套索切片范围裁剪开关 + 宽度 N、多视图套索开关、三视图模式开关

## 二、核心 AI 提示交互（上游功能）

### 2.1 四种提示类型

| 提示 | 快捷键 | 交互方式 | 说明 |
|------|--------|----------|------|
| **点提示（Point）** | `O` | 在切片或 3D 视图放点 | 支持正向（添加）/ 负向（排除）极性；按 `T` 切换极性 |
| **包围盒（BBox）** | `B` | 放两个角点 | 正/负极性；角点放完后自动提交 |
| **套索（Lasso）** | `L` | 闭合曲线 | 在切片上圈选单层区域；按 `Shift+L` 手动提交 |
| **涂鸦（Scribble）** | `S` | 笔刷涂抹 | 基于相邻笔画差分（只传"新增"部分）；正/负极性 |

其他操作：`E` 新建分割段、`R` 清空当前段。

### 2.2 坐标约定

- Slicer 交互坐标为 RAS 世界坐标
- 发往服务端前转为体素坐标 `(x, y, z)` 并反转为 `(z, y, x)` 与 numpy 对齐
- 所有 numpy 掩膜均为 `(z, y, x)` 排列

## 三、切片视图行为（二次开发）

### 3.1 切片交叉线

在三个标准 2D 视图（红/黄/绿）显示其他切面的交线，提供三维位置参照。模块加载时自动启用。

### 3.2 滚轮吸附原始体素网格

- 复选框：`Snap slices to voxel grid`（默认开启）
- 鼠标滚轮每步**恰好前进一张真实切片**，消除插值中间帧的显示模糊
- 自动检测滚动方向，并将偏移量吸附到最近的真实切片位置

### 3.3 斜采集面自动对齐

- 当检测到序列为斜采集（方向余弦偏离最近 RAS 轴 > 2.5°）时，自动将视图旋转到序列自身的采集平面
- 旋正后逐张滚动即为真实切片，交叉线与序列真实轴对齐
- 与滚轮吸附联动：切换背景体积时自动重对齐

## 四、多平面显示体积（二次开发）

**问题背景**：一次检查的多个序列（横断/矢状/冠状）在各自采集平面清晰，但在其他平面因插值而模糊。

**解决方案**：三个 2D 视图（红/黄/绿）的背景可以分别指定不同的已配准序列。

| 控件 | 功能 |
|------|------|
| `Red / Yellow / Green` 下拉选择器 | 为各视图独立指定显示体积 |
| `Apply view volumes` | 应用当前选择；锁定快照（不受后续改选影响）|
| `Use source volume for all` | 恢复所有视图为分割源体积 |

**关键特性**：
- **分割源体积不变**：显示背景与分割几何基准完全解耦，提示坐标始终以分割源体积为基准
- **粘性重应用**：隐藏的 Scribble / Lasso(3D) 编辑器激活时会重置切片背景，插件自动在其后安排一次静默重应用，确保背景不被覆盖
- **快照锁定**：快照在点击 Apply 时锁定，后续修改下拉框不会悄悄生效，直到再次点击 Apply

## 五、原生序列推理（二次开发）

在补充序列（而非主源体积）上运行 AI，结果先进入**预览**，用户确认后再合并。

### 5.1 工作流

1. 启用 `Analyze a supplemental series` → 选择推理工作体积
2. 对任意提示，AI 在工作体积上推理 → 结果重采样到输出网格 → 显示为紫色预览层（不修改主分割）
3. 确认满意后点 `Sync preview to source`，按同步模式合并

### 5.2 同步模式（`Sync mode`）

| 模式 | 效果 |
|------|------|
| **Add**（默认） | 预览掩膜并入主分割（OR 运算），保留已有编辑 |
| **Replace** | 预览替换主分割 |
| **Subtract** | 从主分割中减去预览区域 |
| **Intersect** | 只保留主分割与预览的交集 |

## 六、补充序列自动配准（二次开发）

**问题背景**：两个序列可能来自不同 DICOM 参考坐标系（FrameOfReferenceUID 不同），直接叠加会错位。

**解决方案**：启用补充序列推理或应用多平面背景时，自动检测坐标系差异并异步配准。

| 控件 | 功能 |
|------|------|
| `Auto-register supplemental series to source`（默认开启）| 检测到坐标系差异时自动触发 |
| `I confirm these series are already aligned` | 跳过配准检查 |
| `Registration mode` | 刚性（默认）/ 仿射 |
| `Register now` | 手动触发配准 |
| `Clear alignment` | 清除已挂载的变换 |

**关键特性**：
- 配准使用 BRAINSFit CLI 异步运行；配准期间所有提示和同步操作被阻塞，防止空间错位的交互
- 结果近似单位变换时（平移 < 阈值 且 旋转 < 阈值）视为已对齐，丢弃变换避免无谓重采样
- 变换挂载后切片视图实时跟随，配准完成即生效
- 多个序列的配准任务排队串行执行（一次跑一个 BRAINSFit CLI）
- 变换按 `(补充序列, 源体积)` 对缓存，重复启用直接复用

## 七、高分辨率各向同性输出网格（二次开发）

**问题背景**：DICOM 序列通常层内像素精细（如 0.5 mm）而层间很厚（如 5 mm），直接在源网格存分割会在层间呈现台阶状边界。

**解决方案**：可选的各向同性细网格（与源网格解耦）作为分割的标准存储基准。

| 控件 | 功能 |
|------|------|
| `Store masks on a high-resolution isotropic grid` | 启用/关闭 |
| `Isotropic spacing (mm, 0=auto)` | 目标间距；`0` = 自动取源最细间距 |

**关键特性**：
- 启用后所有掩膜（AI 结果、布尔操作、同步上传）均以此网格为基准
- 内建体素数量预算护栏，超限时自动粗化间距并提示
- 切换间距 / 开关时，当前分割自动重投影到新网格；布尔操作撤销历史同时清空（旧快照几何不再匹配）
- 网格由源体积同向建立，保证后续平滑插值的"共面"前提

---

## 八、平滑插值（二次开发）

**问题背景**：AI 结果落在源网格上，层间仍有台阶；高分辨率网格本身不自动消除台阶。

**解决方案**：把每次 AI 结果写入细网格时做形状感知插值，在层间重建平滑边界。

| 控件 | 功能 |
|------|------|
| `Smooth (interpolate) result between slices` | 启用（自动开启高分辨率输出） |
| `Smooth current segment`（按钮） | 对整个当前分割段手动重跑一次平滑 |

**算法**：对共面网格（源与输出同轴）用**符号距离场（SDF）插值**——计算粗掩膜的有符号距离场并在细网格上用双线性插值重建，零值面即为平滑边界；对非共面情况（补充序列推理结果）退回最近邻重采样 + 高斯平滑。

**作用范围**：仅 AI 提示返回的结果；手动布尔操作、魔棒、ROI 编辑**不经过**平滑，保持精确。

---

## 九、非破坏式显示平滑（二次开发）

不修改底层二值掩膜数据，只在显示层面让分割在 2D/3D 视图渲染得平滑。

| 控件 | 功能 |
|------|------|
| `Display smooth` 复选框 | 开/关 |
| 强度滑块 | 控制 windowed-sinc 平滑强度（0–1）|
| `Bake display smooth`（按钮） | 将平滑后的闭合曲面表示烘焙回 labelmap（导出前使用）；失败自动回滚 |

类比 Blender 的非破坏式修改器：视图里看起来平滑，底层数据不变。需要数据真实平滑时再烘焙。

---

## 十、多序列 SDF 融合（二次开发）

将多个序列的 AI 推理结果 SDF 平均融合为一个平滑联合掩膜。

| 控件 | 功能 |
|------|------|
| `Enable series fusion` 复选框 | 开启后每次 AI 提示结果自动收集 |
| `Fuse N series`（按钮） | 对已收集的 N 个序列结果执行融合 |

**算法**：对每个序列的二值掩膜分别计算有符号距离场，平均后阈值为 0，得到"SDF 均值面"。各序列权重相等。结果写入当前分割段，操作可撤销。

## 十一、选区布尔操作（二次开发）

对已生成的 AI 分割段做局部精修，无需重跑推理。

### 11.1 四种布尔运算

| 运算 | 效果 |
|------|------|
| **Add**（OR） | 将操作数区域并入当前段 |
| **Subtract**（AND NOT） | 从当前段中去除操作数区域 |
| **Intersect**（AND） | 只保留当前段与操作数的交集 |

### 11.2 四种操作数来源

#### ROI（矩形/球/椭球）
- 在 3D 视图放置并调整 ROI 节点
- 支持盒形（轴对齐 OBB）、球形、椭球形，通过 `cbRoiShape` 选择
- 盒形用精确 OBB 包含检测栅格化，对斜采集体积也正确

#### Magic Wand（AI 辅助选区）
- 在视图中放种子点（正向/负向）
- 插件临时备份服务端交互状态 → 重置 → 把种子作为点提示发给 AI → 收到区域掩膜 → 恢复服务端状态（不污染当前推理会话）
- 支持形态学生长/收缩（`sbGrowShrinkWand`，±20 体素）
- 支持 `Preview`（先看效果）再 `Apply`

#### Segment（另一个分割段）
- 从下拉列表选择当前场景中的另一个段作为操作数
- 自动处理参考几何对齐

#### Lasso (3D)（3D 套索区域）
- 点击 `Draw lasso (3D)` 进入绘制模式，在 3D 视图拖拉闭合环
- 黄色区域 = 输入（挤出的实心 3D 块）→ AI Lasso 接口生成精化预览（青色）→ `Apply`
- 同 Magic Wand：先 Preview 确认，再 Apply；支持 Clear Region / Clear Preview

### 11.3 撤销与同步
- 私有撤销栈（最多 10 步），独立于 Slicer Segment Editor 撤销历史
- Apply 后自动同步到服务端，后续 AI 提示基于修改后的分割继续迭代
- 输出网格变化时撤销栈自动清空（状态栏提示）

## 十二、三视图多序列模式（二次开发）

将三个已配准序列（横断/矢状/冠状）整合为一体，在任意视图交互时自动路由到该视图所显示的序列，并在每次交互后方向加权融合显示。

### 12.1 启用方式

1. 在"多平面显示"面板把三个序列分别指给红/黄/绿视图，点 `Apply view volumes`
2. 勾选 `Tri-planar multi-series mode`

`_setup_triplanar_views` 自动启用高分辨率输出 + 序列融合。若分配的不同序列 < 2 个，弹提示要求先设置多平面显示。

### 12.2 按视图路由

放点/框/套索/涂鸦时，插件通过 `_view_for_ras_point` 判断控制点落在哪个切片平面，将该次 AI 推理路由到对应视图的序列：
- **点、框**：取最后一个控制点的 RAS 坐标判断视图
- **套索**：取曲线首点 RAS；三视图模式下不累积多视图套索，逐视图即时路由提交
- **涂鸦**：取笔画质心 RAS 判断视图；diff 掩膜在源网格，自动重采样到路由序列

### 12.3 方向加权融合

每次 AI 结果收集后，`_fuse_series_results` 进行方向加权融合：
1. 对每个序列的掩膜，**仅沿其采集/层叠方向做各向异性高斯平滑**（sigma 由层厚 / 输出间距推算）
2. 各序列 SDF 平均 → 阈值 0 → 写入当前段

**效果**：某方向台阶锯齿被该方向平滑压制，而在该方向清晰的序列主导该方向边界，实现"各方向取最清晰序列主导"。

对 oblique 序列：层叠方向投影到最近输出网格主轴，近似去锯齿（满足大多数采集，强成角可升级精确实现）。

## 十三、套索切片范围裁剪（二次开发）

限制套索结果只保留套索平面附近的指定切片范围，消除 AI 在套索平面外的"溢出"。

| 控件 | 功能 |
|------|------|
| `Enable lasso slice-range clipping` | 开/关 |
| 宽度 N（spinbox） | 保留套索平面中心 ±N 张切片 |

- 高分辨率输出网格激活时自动跳过裁剪（切片索引在源体素坐标下，无法映射到细网格）
- 多视图套索下不应用裁剪（各视图平面不同，单平面裁剪无意义）

---

## 十四、多视图套索（二次开发）

跨多个视图分别绘制套索，将各视图的套索掩膜分别作为独立的 lasso 交互提交给 AI，从多个平面约束 AI 的分割结果。

| 控件 | 功能 |
|------|------|
| `Multi-view lasso` 复选框 | 开启后套索不立即提交，先累积 |
| `Submit all lasso views`（按钮） | 批量提交所有已累积的视图套索 |

**三视图模式开启时，多视图套索自动旁路**（各视图已各自路由到对应序列，不再需要累积）。

---

## 十五、每分割段不透明度（二次开发）

`Segment opacity` 滑块：调整当前选中分割段的不透明度，默认值跨会话持久保存，新建分割段时自动应用该默认值。

## 十六、功能协同与依赖关系

```
三视图多序列模式
  └─ 依赖：多平面显示体积（视图 -> 序列映射）
  └─ 自动启用：高分辨率输出 + 序列融合
  └─ 内部：方向加权融合（_fuse_series_results 升级版）

高分辨率输出网格
  └─ 前提：平滑插值（SDF 插值需要细网格，开平滑自动开高分辨率）
  └─ 影响：布尔操作撤销栈（网格变化即清空）
  └─ 影响：套索切片裁剪（高分辨率开启时跳过裁剪）

补充序列推理（原生序列推理）
  └─ 依赖：多平面显示（共用工作体积面板）
  └─ 前提：自动配准（坐标系不同时需先配准）
  └─ 可选：序列融合（收集每个序列的结果）

选区布尔操作（Magic Wand / Lasso 3D）
  └─ 临时调用 AI 接口（备份/恢复服务端状态）
  └─ 结果通过 show_segmentation → 上传服务端同步
```

## 十七、已知限制

| 限制 | 说明 |
|------|------|
| 服务端单会话 | 多用户并发会互相覆盖 |
| 三视图逐视图重传 | 每次路由到不同序列都需重传图像 + 分割，无法在跨视图的同一会话内迭代细化 |
| Scribble 分辨率 | 涂鸦笔画在源体积分辨率绘制，再重采样到路由序列 |
| 斜采集融合近似 | 强成角序列的方向加权融合使用"近似到最近主轴"方法，极端成角下精度有限 |
| BRAINSFit 依赖 | 自动配准需要 Slicer 内置 BRAINSFit 模块可用；非 DICOM 导入无法读取坐标系 UID，需手动确认对齐 |
| 显示平滑纯视图 | 不烘焙的情况下导出的数据仍为原始 labelmap |
| 高分辨率内存 | 大视野细间距网格内存消耗大；超预算自动粗化间距 |
